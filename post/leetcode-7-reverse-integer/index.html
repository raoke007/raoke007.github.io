Given a 32-bit signed integer, reverse digits of an integer.

Example 1:
  Input: 123
  Output: 321
  
Example 2:
  Input: -123
  Output: -321
	
Example 3:
  Input: 120
  Output: 21
  
Note:
Assume we are dealing with an environment which could only store integers within the 32-bit signed integer range: [−231,  231 − 1]. For the purpose of this problem, assume that your function returns 0 when the reversed integer overflows.

暴力实现法

思路：

  • 由于反转后需要验证数值处于[-2^31, 2^31 - 1]区间，直接用StringBuilder的reverse()方法行不通。
  • 那么就可以在遍历过程中，依次用当前索引的值加上之前索引对应值得和，这里要注意索引的位置（个位、十位、百位...)

class Solution {
    public int reverse(int x) {
        if (x == Integer.MIN_VALUE || x == Integer.MAX_VALUE) {
            return 0;
        }

        int sum = 0;
        if (x >= 0) {
            String strX = x + "";
            int len = strX.length();

            for (int i = 0; i < len; i++) {
                char c = strX.charAt(i);
                if (Integer.parseInt(String.valueOf(c)) * Math.pow(10, i) > Integer.MAX_VALUE) {
                    return 0;
                } else {
                    sum = (int) (Integer.parseInt(String.valueOf(c)) * Math.pow(10, i) + sum);

                    if (sum == Integer.MAX_VALUE) {
                        return 0;
                    }
                }
            }

            return sum;
        } else {
            String strX = Math.abs(x) + "";
            int len = strX.length();

            for (int i = 0; i < len; i++) {
                char c = strX.charAt(i);
                if (Integer.parseInt(String.valueOf(c)) * Math.pow(10, i) > Integer.MAX_VALUE) {
                    return 0;
                } else {
                    sum = (int) (Integer.parseInt(String.valueOf(c)) * Math.pow(10, i) + sum);

                    if (sum == Integer.MAX_VALUE) {
                        return 0;
                    }
                }
            }

            return Integer.parseInt("-" + sum);
        }
    }
}
