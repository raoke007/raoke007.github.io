The Fibonacci numbers, commonly denoted F(n) form a sequence, called the Fibonacci sequence, such that each number is the sum of the two preceding ones, starting from 0 and 1. That is,

F(0) = 0,   F(1) = 1
F(N) = F(N - 1) + F(N - 2), for N > 1.

Given N, calculate F(N).

Example 1:
  Input: 2
  Output: 1
  Explanation: F(2) = F(1) + F(0) = 1 + 0 = 1.
Example 2:
  Input: 3
  Output: 2
  Explanation: F(3) = F(2) + F(1) = 1 + 1 = 2.
Example 3:
  Input: 4
  Output: 3
  Explanation: F(4) = F(3) + F(2) = 2 + 1 = 3.

暴力实现法

思路：

  • 按题意，使用递归即可

class Solution {
    public int fib(int N) {
        if (N == 0) {
            return 0;
        }

        if (N == 1) {
            return 1;
        }

        return fib(N - 1) + fib(N - 2);
    }
}

优化一：

思路：

  • 上述解法在递归过程中，对于相同子问题出现多次计算情况，如下图

  • 如果在每次递归过程中，每个子问题的计算结果都保存下来，这样经过判断就可以保证每次子问题只计算一次，这种解决思路叫记忆化搜索

class Solution {
    private int[] nums;
    
    public int fib(int N) {
        this.nums = new int[N + 1];
        for (int i = 0; i < N + 1; i++) {
            this.nums[i] = -1;
        }
        
        return calculateWays(N);
    }
    
    private int calculateWays(int N) {
        if (N == 0) {
            return 0;
        }

        if (N == 1) {
            return 1;
        }

        if (this.nums[N] == -1) {
            this.nums[N] = calculateWays(N - 1) + calculateWays(N - 2);
        }
        
        return this.nums[N];
    }
}

优化二：

思路：

  • 上述的递归解决方案是自顶而下的解决问题，递归是假设子问题已经被解决。同时加入记忆化搜索

  • 此时也可以使用动态规划的思想来解决此问题，动态规划是自底向上的解决问题

动态规划：将原问题分解成若干子问题，同时保存每个子问题的答案，使得每个子问题只被计算一次，最终获得原问题的答案

class Solution {
    
    public int fib(int N) {
        if (N == 0) {
            return 0;
        }
        
        if (N == 1) {
            return 1;
        }
        
        int[] nums = new int[N + 1];
        nums[0] = 0;
        nums[1] = 1;
        
        for (int i = 2; i <= N; i++) {
            nums[i] = nums[i - 1] + nums[i - 2];
        }
        
        return nums[N];
    }
}
